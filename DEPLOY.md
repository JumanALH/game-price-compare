# 🚀 كيف تنشر الموقع للعامّة (مجانًا)

الموقع فيه خادم (Node)، فما ينفع نرفعه على استضافة ملفات عادية.
راح نستخدم **Render** — مجاني، بدون بطاقة، وينشر مباشرة من GitHub.

الفكرة: نرفع الكود على **GitHub** → نربطه بـ **Render** → يعطينا رابط عام.

---

## الجزء الأول: رفع المشروع على GitHub

> بدون أوامر — كله من الموقع.

1. سجّل دخول على <https://github.com> (عندك حساب: JumanALH).
2. اضغط **New** (زر أخضر) لإنشاء مستودع جديد.
3. اكتب اسم: مثلًا `game-price-compare`، خله **Public**، واضغط **Create repository**.
4. في الصفحة الجديدة اضغط رابط **uploading an existing file**.
5. اسحب ملفات المشروع كلها داخل الصندوق:
   - `server.js`
   - `package.json`
   - `package-lock.json`
   - `README.md`
   - `.gitignore`
   - ومجلد `public` كامل (فيه `index.html`)
   - ⚠️ **لا ترفع مجلد `node_modules`** (لو موجود، تجاهله — Render يبنيه بنفسه).
6. اضغط **Commit changes**. خلص، الكود صار على GitHub.

---

## الجزء الثاني: النشر على Render

1. افتح <https://render.com> واضغط **Get Started** → سجّل بحساب **GitHub**
   (أسهل طريقة عشان يوصل لمستودعاتك).
2. من اللوحة اضغط **New +** → اختر **Web Service**.
3. اختر مستودع `game-price-compare` من القائمة → **Connect**.
4. عبّي الإعدادات كذا:

   | الحقل | القيمة |
   |------|--------|
   | **Name** | أي اسم (يصير جزء من الرابط) |
   | **Region** | الأقرب لك (مثلًا Frankfurt) |
   | **Branch** | `main` |
   | **Runtime** | Node |
   | **Build Command** | `npm install` |
   | **Start Command** | `npm start` |
   | **Instance Type** | **Free** |

5. اضغط **Create Web Service** وانتظر شوي لين يخلّص البناء (Build).
6. لما يطلع **Live**، بيعطيك رابط شكله:
   ```
   https://game-price-compare.onrender.com
   ```
   هذا رابطك العام — أرسله لأي أحد 🎉

## ⏰ إبقاء الموقع صاحي (ما ينام)

الخطة المجانية في Render تنوّم الموقع بعد ١٥ دقيقة بدون زيارات.
عندك **حلّين** (والأفضل تفعّل الاثنين):

### 1) مدمج بالموقع نفسه ✅ (شغّال تلقائيًا)

الخادم فيه ميزة **Keep-alive** جاهزة: على Render يكتشف رابطه العام تلقائيًا
(من متغير `RENDER_EXTERNAL_URL`) ويزور نفسه على `/health` كل ١٠ دقايق،
فما يوصل حد الـ ١٥ دقيقة وينام. **ما تحتاج تسوي شي** — تشتغل من أول نشرة.

> لو استخدمت استضافة غير Render، أضف متغير بيئة `SELF_URL` بقيمة رابط موقعك
> ونفس الميزة بتشتغل.

### 2) UptimeRobot (طبقة أمان إضافية — موصى فيه)

الـ self-ping ينقطع لو الخدمة أعيد تشغيلها وهي نايمة، فالأضمن خدمة خارجية:

1. سجّل مجانًا على <https://uptimerobot.com>.
2. اضغط **+ New Monitor**.
3. عبّيها كذا:
   - **Monitor Type**: `HTTP(s)`
   - **Friendly Name**: أي اسم (مثلًا GamePrice)
   - **URL**: رابط موقعك + `/health` — مثال:
     `https://game-price-compare.onrender.com/health`
   - **Monitoring Interval**: كل **5 دقايق**
4. اضغط **Create Monitor**. خلص ✅ — UptimeRobot بيزور موقعك كل ٥ دقايق
   فيبقى صاحي دايم، وكمان ينبهك بإيميل لو الموقع طاح.

## ملاحظات مهمة

- **التحديثات**: أي تعديل ترفعه على GitHub، Render ينشره تلقائيًا.
- **GOG**: لو ضغط كثير يحجب مؤقتًا، والموقع يتعامل معها (يعرض ستيم ويكمّل).
  التخزين المؤقت يخفف هالشي.
- **الحدود المجانية**: ٧٥٠ ساعة تشغيل بالشهر — تكفي لتشغيل الموقع ٢٤/٧
  (الشهر فيه ~٧٣٠ ساعة). مع الـ keep-alive بتستهلكها كلها وهذا طبيعي.

## لو تبي طريقة أسرع بأوامر Git (اختياري)

لو مثبت Git على جهازك، من داخل مجلد المشروع:

    git add .
    git commit -m "GamePrice v3"
    git push

بعدها Render ينشر التحديث تلقائيًا.
